# 第一章：基础概念入门

## 本章简介

本章是 MPI 学习的起点，带你理解并行计算的核心概念和 MPI 的定位。

## 为什么要学并行计算？

### 单核计算的瓶颈

想象一下，你有一堆沙子需要数清楚。一个人在沙堆前一颗一颗数，可能需要几天几夜。但如果叫 100 个人同时数，最后把结果加起来，只需要几分钟。

这就是**并行计算**的核心思想：**用多个计算单元同时工作，解决单个单元无法快速完成的问题**。

### 现代计算面临的挑战

- **计算量巨大**：气候模拟需要计算数十亿网格点
- **数据规模爆炸**：深度学习模型参数达千亿
- **时间要求严格**：实时预测不能等几天

### 并行计算的两大流派

| 类型 | 共享内存 | 分布式内存 |
|------|----------|------------|
| 典型技术 | OpenMP, Pthreads | MPI |
| 内存关系 | 所有核共享同一内存 | 每个核有独立内存 |
| 扩展性 | 单节点，多核 | 多节点，超大规模 |
| 通信方式 | 读写共享变量 | 发送消息 |

## 什么是 MPI？

### 定义

**MPI** = **Message Passing Interface**（消息传递接口）

它不是一个编程语言，而是一套**标准化的通信接口**。就像 USB 接口一样，不管是什么品牌的鼠标、键盘，只要符合 USB 标准就能插上用。MPI 也是如此，不管你用 OpenMPI 还是 MPICH，只要符合 MPI 标准，你的程序就能跑。

### MPI 能做什么？

```
进程 0 (CPU/GPU)      进程 1 (CPU/GPU)      进程 2 (CPU/GPU)
    ↓                    ↓                    ↓
   [内存]  ←───────  [消息传递]  ──────→  [内存]
```

- 让多个进程**同时运行**
- 让进程之间**相互通信**
- 让多个计算单元**协同工作**

### MPI 程序长什么样？

```c
#include <mpi.h>
#include <stdio.h>

int main(int argc, char* argv[]) {
    // 1. 初始化 MPI
    MPI_Init(&argc, &argv);
    
    // 2. 获取进程信息
    int rank, size;
    MPI_Comm_rank(MPI_COMM_WORLD, &rank);  // 当前进程 ID (0, 1, 2, ...)
    MPI_Comm_size(MPI_COMM_WORLD, &size);  // 总进程数
    
    // 3. 通信计算
    printf("Hello from process %d of %d\n", rank, size);
    
    // 4. 结束 MPI
    MPI_Finalize();
    return 0;
}
```

## 核心概念详解

### 1. 通信域 (Communicator)

通信域是 MPI 中的**组织单位**，它定义了一组可以相互通信的进程。

- `MPI_COMM_WORLD`：最常用的通信域，包含所有进程
- 通信域决定了哪些进程可以相互发消息

### 2. 进程秩 (Rank)

- 每个进程在通信域中的**唯一 ID**
- 从 0 开始编号：`0, 1, 2, 3, ...`
- 通过 `MPI_Comm_rank()` 获取

### 3. 消息 (Message)

消息 = 数据 + 目的地 + 标签

- **数据**：要传递的内容（整数、浮点数、数组等）
- **目的地 (dest)**：发给哪个进程
- **标签 (tag)**：消息的"门牌号"，用于匹配发送和接收
- **发送者 (source)**：谁发的消息

### 4. 点对点 vs 集合通信

| 类型 | 说明 | 示例 |
|------|------|------|
| 点对点 | 两个进程之间通信 | MPI_Send, MPI_Recv |
| 集合通信 | 一组进程共同参与 | MPI_Bcast, MPI_Reduce |

## MPI 发展历程

### 版本演进

- **MPI-1 (1994)**：基础功能，点对点 + 集合通信
- **MPI-2 (1996)**：动态进程、并行 I/O、单侧通信
- **MPI-3 (2012)**：非阻塞集合通信、改进的 RMA
- **MPI-4 (2021)**：GPU 支持、灵活拓扑

### 主流实现

| 实现 | 特点 | 适用场景 |
|------|------|----------|
| **OpenMPI** | 开源、社区活跃 | 默认首选 |
| **MPICH** | 轻量、高性能 | 性能敏感场景 |
| **Intel MPI** | Intel 硬件优化 | Xeon 集群 |

## 常见问题

**Q: MPI 和 OpenMP 有什么区别？**
A: MPI 是分布式内存（多台机器），OpenMP 是共享内存（单台机器多核）。实际常混合使用。

**Q: MPI 程序能跑多快？**
A: 理论上限是单核的 N 倍（N = 进程数），实际受通信开销、数据交换等因素影响。
